API Documentation

AI-powered job application automation

Getting Started

The Agent Stapply API enables automated job applications using AI-powered browser automation. The API accepts job details and candidate information, then autonomously navigates application forms, fills in data, and submits applications.

Base URL

https://your-api-domain.com
Authentication

API authentication will be required for production use. Contact your administrator for API credentials.

Endpoints

POST /apply

Start an automated job application process. Returns a session ID immediately and runs the agent asynchronously.

Request Body

Parameter Type Description
user_idrequired string Unique identifier for the user
urlrequired string Target job application URL
resume_urlrequired string URL to the candidate's resume file
profileoptional object Candidate profile data for form filling
instructionsoptional string Additional instructions for the agent
secretsoptional object Sensitive data to pass to the agent (not logged)
webhook_urloptional string Webhook URL for completion notification (defaults based on environment)
modeloptional string Custom LLM model (format: openai/model-name, google/model-name, or groq/provider/model-name)

Example Request

{ "user_id": "user_123", "url": "https://company.com/careers/apply", "resume_url": "https://storage.example.com/resumes/user_123.pdf", "profile": { "first_name": "John", "last_name": "Doe", "email": "john@example.com", "phone": "+1234567890" }, "instructions": "Apply for the software engineer position", "model": "openai/gpt-4" }

Response

{ "session_id": "session_abc123", "live_url": "https://anchor.browser/live/session_abc123" }

Response Fields

Field Type Description
session_id string Unique identifier for the application session
live_url string URL to watch the browser automation live

Model Providers

The API supports multiple LLM providers. Specify the model using the format below:

OpenAI

"model": "openai/gpt-4" "model": "openai/gpt-4-turbo" "model": "openai/gpt-3.5-turbo"

Google Gemini

"model": "google/gemini-pro" "model": "google/gemini-1.5-pro"

Groq

"model": "groq/anthropic/claude-3.5-sonnet" "model": "groq/meta/llama-3.1-70b"
Default Model

If no model is specified, the system uses the default configured model.

Webhooks

When an application completes, the API sends a POST request to the configured webhook URL with the following payload:

Webhook Payload

{ "session_id": "session_abc123", "success": true, "user_id": "user_123", "agent_result": { "duration_seconds": 45.2, "final_result": "Application submitted successfully", "success": true, "has_errors": false }, "cost_metadata": { "total_tokens": 15420, "total_cost": 0.234 }, "timestamp": 1729886400 }

Webhook Security

Webhook requests include HMAC-SHA256 signatures in the X-Webhook-Signature header. Verify the signature using your webhook secret to ensure requests are from the API.

Error Responses

The API returns standard HTTP status codes:

Status Code Description
200 Success
202 Accepted - Request received and processing started
400 Bad Request - Invalid parameters
403 Forbidden - Unauthorized origin or access denied
500 Internal Server Error - Something went wrong

Error Response Format

{ "error": "Bad Request", "detail": "Missing required field: user_id" }
